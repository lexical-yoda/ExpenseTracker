Email Automation (n8n)
Automatically parse bank transaction emails using n8n — a free, self-hosted automation tool that connects your email inbox to this app.

How it works:

Bank sends email → Your inbox → n8n detects it → n8n sends it to this app → LLM parses it → Draft appears for review

▶ Setup Guide
  1. Install n8n

    Option A — Quick start:

    docker run -d --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n n8nio/n8n

    Option B — Docker Compose (recommended):

    services: n8n: image: n8nio/n8n:latest container_name: n8n ports: - "5678:5678" volumes: - n8n_data:/home/node/.n8n environment: - N8N_SECURE_COOKIE=false restart: unless-stopped volumes: n8n_data:

    Then open http://localhost:5678 and create an account.

  2. Download the workflow template
    Download n8n Workflow Template
  3. Import into n8n
    In n8n: click + Add Workflow → Import from File → select the downloaded JSON file.
  4. Configure Node 1: Email Trigger
    Click the "Email Trigger (IMAP)" node → Add credentials:
    Gmailimap.gmail.com : 993 (SSL) + App Password
    Outlookoutlook.office365.com : 993 (SSL) + App Password
    Yahooimap.mail.yahoo.com : 993 (SSL) + App Password
    Protonmail127.0.0.1 : 1143 (STARTTLS) via Protonmail Bridge
  5. Configure Node 2: Filter Bank Emails
    Replace REPLACE_WITH_BANK_SENDER_EMAIL with your bank's alert email address (e.g., alerts@hdfcbank.net).
  6. Configure Node 3: Send to Expense Manager
    Replace these values (copy from the Webhook section below):
    • REPLACE_WITH_APP_URL → your app URL (e.g., https://expenses.yourdomain.com)
    • REPLACE_WITH_API_KEY → your API key
  7. Activate the workflow
    Toggle the switch in the top-right of the n8n workflow editor. Done!

Custom Parsing Prompt
Customize the system prompt sent to the LLM for parsing bank emails.
▶ Prompt Setup Guide — create a prompt for your bank

How to create a parsing prompt for your bank:

  1. Copy 3-4 different transaction alert emails from your bank (include: debit, credit, UPI, card transactions)
  2. Open any capable LLM (ChatGPT, Claude, Gemini, etc.)
  3. Paste this prompt along with your sample emails:
    I have a self-hosted expense tracker that parses bank transaction emails using an LLM. I need you to create a system prompt that extracts transaction details from my bank's email alerts. Here are sample emails from my bank: [paste your emails here] The output must be a JSON object with exactly these fields: - amount (number, always positive) - merchant (string, clean merchant name) - date (string in YYYY-MM-DD format) - account (string, must match one of my account names listed below) - category (string, best guess from: Groceries, Dining, Transport, Utilities, Shopping, Health, Entertainment, Education, Miscellaneous) - type (string, either "Expense" or "Income") My accounts are: [list your account names here] Create a system prompt with explicit rules for: - Parsing my bank's specific date formats - Extracting merchant names (removing bank-specific prefixes) - Mapping account identifiers to my account names - Handling the current year correctly The prompt should instruct the LLM to return ONLY a JSON object, no explanation or markdown.
  4. Copy the generated system prompt into the text area above
  5. Use "Test Parse" below to verify it works
